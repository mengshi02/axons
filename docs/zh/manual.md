## 项目

### 创建项目

通过「创建项目」功能，可以在 Axons 中新建一个空白项目，并配置项目的基本信息。

![Axons 欢迎界面](screenshot/01-project/01-welcome.png)

> 首次打开 Axons 时，会看到如上图所示的欢迎界面。页面中央提供了「导入本地路径或远程 URL」的入口，支持拖拽文件夹或点击浏览来快速导入项目。界面下方展示了 Axons 的三大核心能力：可视化交互图谱、代码搜索导航和 AI 智能体。

**操作步骤：**

1. 在左侧导航栏点击「项目」入口。
2. 点击「创建项目」按钮。
3. 在弹出的对话框中填写项目名称、描述等基本信息。
4. 选择项目的存储路径。
5. 点击「确认」完成创建。

创建完成后，系统将自动跳转至新项目的工作空间，并开始初始化代码图谱。

### 导入项目

通过「导入项目」功能，可以将已有的本地代码仓库导入到 Axons 进行分析。Axons 支持两种导入方式：本地路径和远程 URL。

#### 从本地路径导入

![从本地路径导入项目](screenshot/01-project/02-import-local.png)

> 选择「Local」标签页后，在「Project Path」输入框中填写本地项目目录的绝对路径（例如 `/users/you/myproject`）。勾选「Auto-watch file changes」可让 Axons 在代码文件变更时自动更新图谱。填写完成后点击「Import」即可开始导入。

**操作步骤：**

1. 在左侧导航栏点击「项目」入口。
2. 点击「导入项目」按钮，弹出导入对话框。
3. 选择「Local」标签页。
4. 在「Project Path」中填写本地仓库的绝对路径。
5. 按需勾选「Auto-watch file changes」。
6. 点击「Import」开始构建图谱。

#### 从远程 URL 导入

![从远程 URL 导入](screenshot/01-project/03-import-remote-url.png)

> 选择「Remote URL」标签页后，在「Repository URL」中填写 GitHub 仓库地址（支持 HTTPS 和 SSH 格式）。Axons 会自动将仓库克隆到本地并进行分析。

![远程导入高级选项](screenshot/01-project/04-import-advanced-options.png)

> 展开「Advanced options」后，可以指定克隆的分支（默认 `main`）以及克隆位置。选择「Managed」时，Axons 会将仓库克隆到统一管理目录 `~/.axons/repos/`；选择「Custom」可自定义本地路径。配置完成后点击「Clone & Import」。

**操作步骤：**

1. 选择「Remote URL」标签页。
2. 填写 GitHub 仓库地址（支持 HTTPS 和 SSH 格式）。
3. 按需展开「Advanced options」配置分支和克隆路径。
4. 勾选「Auto-watch file changes」（可选）。
5. 点击「Clone & Import」，等待克隆和图谱构建完成。

导入过程中，系统将自动解析代码结构、构建依赖关系图谱，完成后可在图谱面板中查看全貌。

## 设置

### 系统设置

系统设置提供全局级别的配置选项，对所有项目生效。

#### 主题设置

![主题设置](screenshot/02-settings/01-theme.png)

> 在「Theme」标签页中，可以在深色「Moon」主题（深色背景配紫色强调色，适合夜间使用）和浅色「Sun」主题（明亮风格，适合白天使用）之间切换。点击对应主题卡片即可实时预览并应用。

#### Embedding 设置

![Embedding 设置](screenshot/02-settings/02-embedding.png)

> 在「Embedding」标签页中，可以配置向量嵌入模型，用于支持代码语义搜索功能。勾选「Enable auto-embedding after build」后，每次图谱构建完成时会自动生成嵌入向量。可选择 Provider（如 Custom），并填写 API Key、模型名称和 Base URL。

#### LLM 模型设置

![LLM 模型设置](screenshot/02-settings/03-llm.png)

> 在「LLM」标签页中，可以管理 AI 面板所使用的大语言模型。勾选「Enable LLM Agent」后，AI 智能体将可以回答代码相关问题。列表中展示了已配置的模型（如 Gemma、Qwen 等自定义模型），点击「+ Add Model」可添加新模型。

#### Rerank 设置

![Rerank 设置](screenshot/02-settings/04-rerank.png)

> 在「Rerank」标签页中，可以启用搜索结果重排序功能以提升语义搜索的精确度。勾选「Enable reranking for search results」后，选择 Provider（如 Cohere）并填写 API Key，即可启用重排序。

#### RAG 设置

![RAG 设置](screenshot/02-settings/05-rag.png)

> 在「RAG」标签页中，可以配置检索增强生成（RAG）管道的参数。主要参数包括：Chunk size（文本分块大小，默认 1000）、Chunk overlap（分块重叠量，默认 200）和 Top K Results（检索返回结果数量，默认 10）。勾选「Enable reranking in RAG pipeline」可对 RAG 结果进行二次排序，提升回答质量。

**可配置项包括：**

| 配置项 | 说明 |
|--------|------|
| 主题 | 切换界面明暗主题 |
| 语言 | 设置界面显示语言 |
| Embedding | 配置向量嵌入模型 |
| LLM | 管理大语言模型 |
| Rerank | 配置搜索重排序 |
| RAG | 配置检索增强生成参数 |

**操作入口：** 点击顶部菜单栏或侧边栏的「设置」图标，选择「系统设置」。

---

### 项目设置

项目设置仅对当前项目生效，可针对不同项目进行差异化配置。

**可配置项包括：**

| 配置项 | 说明 |
|--------|------|
| 项目名称 | 修改项目的显示名称 |
| 根目录 | 调整项目扫描的根路径 |
| 忽略规则 | 配置不参与分析的文件或目录（支持 glob 语法） |
| 分析语言 | 指定项目的主要编程语言 |
| 图谱刷新策略 | 设置代码变更后图谱自动重建的时机 |
| 架构规则 | 绑定当前项目适用的架构规则集 |

**操作入口：** 在项目工作空间内，点击「设置」→「项目设置」。

## 图谱面板

图谱面板是 Axons 的核心可视化区域，以节点和边的形式展示代码中各模块、文件、函数之间的依赖关系。

### 图谱缩放

![图谱全局视图](screenshot/03-graph/01-graph-overview.png)

> 上图展示了加载完成的代码图谱全局视图。图谱中每个圆形节点代表一个代码元素（文件夹、文件、函数或变量），节点间的连线表示依赖或调用关系。右上角显示当前图谱的节点数（3117 nodes）和边数（6447 edges）。底部工具栏提供 Health（代码健康）、Analytics（图分析）、Impact（影响分析）、CFG（控制流图）、Sequence（序列图）、Rules（架构规则）、Flow（执行流）等分析功能入口。

图谱面板支持多种缩放方式，便于在全局视图与局部细节之间切换。

**缩放操作：**

| 操作 | 说明 |
|------|------|
| 鼠标滚轮向上/向下 | 放大 / 缩小图谱 |
| 触控板双指捏合 | 缩放图谱 |
| `Ctrl + +` / `Ctrl + -` | 键盘快捷键放大 / 缩小 |
| `Ctrl + 0` | 重置缩放比例至默认 |
| 工具栏缩放按钮 | 点击图谱面板工具栏中的 `+` / `-` 按钮 |
| 「适应窗口」按钮 | 一键将图谱缩放至完整显示在当前视口 |

---

### 节点操作

图谱中的每个节点代表一个代码元素（模块、文件、类、函数等），支持多种交互操作。

![图谱放大局部视图](screenshot/03-graph/02-graph-zoomed.png)

> 放大图谱后，可以清晰看到节点的图例说明：不同颜色和形状区分 Folder（文件夹）、File（文件）、Class（类）、Function（函数）、Variable（变量）等不同类型的代码元素。节点间的连线方向表示依赖方向。

#### 文件树视图

![图谱文件树视图](screenshot/03-graph/03-graph-filetree.png)

> 在左侧搜索面板中，可以以文件树形式浏览整个项目结构。支持「Node」（节点）和「Full Text」（全文）两种搜索模式。点击树节点可以在图谱中定位到对应位置，展开文件夹可查看子文件。

#### 展开节点依赖

![展开节点查看子依赖](screenshot/03-graph/04-graph-expand-node.png)

> 双击图谱中的目录节点，可以展开查看其内部的文件和子目录。如上图展示了展开 `conf` 目录后，内部所有配置文件（如 `nginx.conf`、`mime.types` 等）以独立节点形式呈现在图谱中，便于详细分析该模块的依赖关系。

**基础操作：**

| 操作 | 说明 |
|------|------|
| 单击节点 | 选中节点，右侧面板显示详情 |
| 双击节点 | 展开 / 折叠节点的子级依赖 |
| 右键节点 | 弹出上下文菜单，提供更多操作选项 |
| 拖拽节点 | 调整节点在图谱中的位置 |
| 悬停节点 | 显示节点的简要信息提示 |

**上下文菜单选项：**

- **在代码面板中打开** — 跳转至该节点对应的源码位置
- **查看影响分析** — 分析该节点变更对整个项目的影响范围
- **查看控制流图** — 展示该函数的控制流结构
- **查看序列图** — 展示该函数的调用时序
- **固定 / 取消固定** — 将节点锁定在当前位置，防止图谱重新布局时移动
- **隐藏节点** — 从当前视图中隐藏该节点

## 代码面板

代码面板提供源码查看与浏览功能，与图谱面板联动，方便在可视化视图与代码细节之间快速切换。

![代码面板与引用查找](screenshot/04-code/01-code-panel.png)

> 上图展示了代码面板的主要功能。在图谱中点击 `ngx_http_file_cache_update` 函数节点后，代码面板右侧自动打开了「References（引用）」面板，展示该函数在项目中的 11 处引用位置，每处引用均标注了文件路径和行号。左侧代码区域高亮显示了函数的源码内容，支持语法高亮，便于快速理解代码逻辑。

**主要功能：**

- **代码高亮** — 支持多种编程语言的语法高亮显示。
- **图谱联动** — 在图谱中选中节点后，代码面板自动定位并高亮对应的代码片段。
- **跳转定义** — 点击代码中的符号，可跳转至其定义位置（支持跨文件跳转）。
- **查找引用** — 右键点击符号，选择「查找所有引用」，展示该符号在项目中的全部使用位置。
- **面包屑导航** — 代码面板顶部显示当前文件的路径层级，支持快速跳转到上级目录。
- **代码折叠** — 支持对函数体、代码块进行折叠，便于查看整体结构。
- **迷你地图** — 面板右侧提供代码缩略图，用于快速定位代码位置。

## 分析面板

分析面板集成了多种代码分析能力，帮助开发者深入理解代码质量与结构。

### 代码健康

代码健康功能对项目进行全面的质量扫描，生成可视化健康报告。

![代码健康入口](screenshot/05-analysis/01-code-health.png)

> 在图谱面板底部工具栏点击「Health」标签，即可进入代码健康分析面板。图谱左侧文件树区域同时展示了项目的整体结构（522 files），底部显示可切换到不同分析维度。

![代码健康热点报告](screenshot/05-analysis/09-code-health-hotspots.png)

> 代码健康分析完成后，面板将展示「Hotspots（热点）」报告。热点视图以可视化方式标注出项目中复杂度高、修改频繁或耦合严重的代码区域，帮助开发者快速定位需要重点关注的模块。

**报告包含以下维度：**

- **复杂度** — 函数圈复杂度统计，识别高复杂度函数
- **重复代码** — 检测项目中相似或重复的代码片段
- **代码耦合** — 分析模块间的耦合程度，识别高耦合区域
- **注释覆盖率** — 统计核心模块的注释比例
- **测试覆盖率** — 结合测试报告展示代码覆盖情况（需配置测试结果路径）

**操作步骤：**

1. 打开「分析面板」，切换至「代码健康」标签页。
2. 点击「开始分析」，等待扫描完成。
3. 查看各维度评分及问题列表。
4. 点击具体问题条目，可跳转至代码面板定位对应代码。

### 图分析

图分析功能基于代码图谱，提供结构层面的深度洞察。

![图分析 - PageRank 指标](screenshot/05-analysis/08-graph-analytics.png)

> 在底部工具栏点击「Analytics」进入图分析面板。上图展示了 PageRank 分析结果——系统计算了图谱中 7,201 个节点的 PageRank 分值，分值越高表示该节点被其他节点依赖的程度越高（即核心程度越高）。通过此指标可快速识别项目中最关键的模块。

**包含分析项：**

- **入度 / 出度排行** — 找出被依赖最多（入度高）或依赖外部最多（出度高）的节点
- **关键路径** — 识别图谱中连接核心模块的关键依赖链路
- **孤立节点** — 检测无任何依赖关系的孤立代码元素
- **循环依赖** — 检测并展示代码中存在的循环依赖关系
- **模块聚类** — 基于依赖关系自动对代码模块进行聚类归组

### 影响分析

影响分析用于评估对某个代码元素进行修改后，可能影响到的范围。

![影响分析视图](screenshot/05-analysis/07-impact-analysis.png)

> 在底部工具栏点击「Impact」进入影响分析面板。系统展示了「Change Impact」（变更影响）和「Call Chain」（调用链）两个维度。高亮路径标注了从目标节点出发、所有会受到影响的下游依赖链路，让开发者在修改代码前充分评估影响范围。

**操作步骤：**

1. 在图谱面板中右键目标节点，选择「查看影响分析」；或在分析面板中手动选择目标节点。
2. 系统将高亮展示所有直接和间接依赖该节点的代码路径。
3. 影响范围以层级列表或图谱高亮两种形式呈现。
4. 可导出影响分析报告，用于代码评审或变更评估。

### 控制流图

控制流图（CFG）以图形化方式展示函数内部的执行路径与分支结构。

![控制流图](screenshot/05-analysis/06-control-flow.png)

> 在底部工具栏点击「CFG」进入控制流图面板。上图展示了 `ngx_stream_get_va...` 函数的控制流图，自动识别并标注了函数中的 If/Else 分支、Loop 循环和 Return 语句，以有向图形式展示各代码块之间的执行跳转路径，帮助理解函数的执行逻辑。

**功能说明：**

- 自动解析函数中的条件分支、循环、异常处理等结构
- 以有向图形式展示各代码块之间的执行跳转关系
- 支持高亮显示特定执行路径

**操作步骤：**

1. 在代码面板或图谱面板中选中目标函数。
2. 右键选择「查看控制流图」，或在分析面板切换至「控制流图」标签页。
3. 在展示的控制流图中，可点击各节点查看对应代码片段。

### 序列图

序列图以时序方式展示函数调用链中各模块之间的交互过程。

![序列图生成面板](screenshot/05-analysis/05-sequence-diagram.png)

> 在底部工具栏点击「Sequence」进入序列图面板。在「Function / Entry Point」输入框中填写目标函数名（如 `handleBuild`、`main`、`processRequest` 等），然后配置「Call Depth」（调用深度，默认为 3），点击「Generate」即可生成该函数的 UML 序列图。

**功能说明：**

- 自动追踪函数的调用链路，生成标准 UML 序列图
- 展示调用方、被调用方及消息传递顺序
- 支持多层级调用链展开

**操作步骤：**

1. 选中目标函数节点。
2. 右键选择「查看序列图」，或在分析面板切换至「序列图」标签页。
3. 查看生成的时序交互图，点击各调用节点可定位到对应代码。

### 架构规则

架构规则功能允许用户自定义代码架构约束，并对项目进行合规性检查。

![架构规则面板 - 空规则](screenshot/05-analysis/03-arch-rules.png)

> 在底部工具栏点击「Rules」进入架构规则面板。初始状态下，Rules（规则）和 Violations（违规）列表均为空。点击「+ Add Rule」可以开始添加架构约束规则。

![架构规则面板 - 执行校验](screenshot/05-analysis/04-arch-rules-validate.png)

> 添加完规则后，点击「Validate」按钮执行架构合规性检查。系统将根据定义的规则扫描整个代码图谱，并在 Violations 列表中列出所有违规的依赖关系，帮助团队及时发现架构退化问题。

**功能说明：**

- 支持定义模块间的禁止依赖规则（如「UI 层不得直接调用 DB 层」）
- 支持定义强制依赖规则（如「所有 Service 必须经由 Interface 调用」）
- 规则使用声明式语法编写，支持导入 / 导出规则集
- 检查结果以违规列表形式展示，并可关联至图谱中的具体依赖路径

**操作步骤：**

1. 打开「分析面板」，切换至「架构规则」标签页。
2. 点击「新建规则」或导入已有规则集文件。
3. 编辑规则内容并保存。
4. 点击「执行检查」，查看违规报告。

### 执行流

执行流功能用于追踪程序在运行时的动态调用链路，辅助理解复杂的执行逻辑。

![执行流分析列表](screenshot/05-analysis/02-execution-flow.png)

> 在底部工具栏点击「Flow」进入执行流面板。上图展示了系统自动识别的 75 个执行流程，每个流程以「入口函数 → 关键调用链 → 步骤数」的形式呈现。例如：`ngx_worker_thread → ngx_timeofday → ...（19 steps）`，`ngx_http_mp4_handler → ngx_http_discard_re...（17 steps）`。点击具体流程可在图谱中高亮显示完整执行路径。

**功能说明：**

- 以有向图形式展示从入口函数出发的完整调用路径
- 支持设置追踪深度，避免调用链过深导致视图过于复杂
- 可结合运行时日志数据进行热点路径高亮

**操作步骤：**

1. 在分析面板切换至「执行流」标签页。
2. 选择入口函数作为分析起点。
3. 配置追踪深度和过滤条件。
4. 点击「生成执行流」，查看调用链路图。

## 终端

Axons 内置了集成终端，支持在不切换工具的情况下直接执行命令行操作。

### 一般功能

![终端 - 单项目视图](screenshot/07-terminal/01-terminal-single.png)

> 上图展示了 Axons 内置终端在单项目模式下的界面。终端面板位于界面底部，当前工作目录为 nginx 项目根目录（`/users/mengshi3/c/nginx`）。图谱面板与终端面板可同时显示，开发者无需离开 Axons 即可执行 shell 命令。

**基础操作：**

| 操作 | 说明 |
|------|------|
| 打开终端 | 点击底部状态栏的「终端」图标，或使用快捷键 `` Ctrl + ` `` |
| 输入命令 | 在命令行提示符后输入命令并按回车执行 |
| 清空终端 | 输入 `clear` 命令或使用快捷键 `Ctrl + L` |
| 复制内容 | 选中终端文本后，使用 `Ctrl + C`（macOS：`Cmd + C`）复制 |
| 粘贴内容 | 使用 `Ctrl + V`（macOS：`Cmd + V`）粘贴 |
| 中断命令 | 使用 `Ctrl + C` 中断当前正在执行的命令 |
| 历史命令 | 使用方向键 `↑` / `↓` 浏览历史命令 |

**工作目录：**

终端默认的工作目录为当前项目的根目录，可通过 `cd` 命令切换。
